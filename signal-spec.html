Methodology

how we score decisions (receipts, not vibes)

This page shows the math behind brief outputs. It is intentionally focused on buyer-facing signals and one worked example.

The 5 Signals On Briefs

momentum

How fast format performance is moving right now.

breadth

How wide creator adoption is in the active window.

saturation risk

How close a format is to compression from replication density.

gate health

How many candidates are passing quality and reliability filters.

reception

How audience response scores on sentiment, objections, proof demand, and purchase intent.

Core Equations

Readable formulas used in production scoring. Variables are standardized per market and time window.

velocity and momentum

vph_views = (ln(v_t + 1) - ln(v_{t-delta} + 1)) / delta_hours
accel_score = 0.50*vph_views + 0.30*vph_shares + 0.20*vph_saves
momentum_score = 100 * sigmoid(1.15*accel_score + 0.35*delta_score)

Velocity rewards acceleration, not only scale.

breadth and saturation

breadth_score = 100 * sigmoid(z(unique_creators_72h) + 0.60*z(unique_posts_72h))
replication_ratio = posts_72h / max(unique_creators_72h, 1)
saturation_risk = 100 * sigmoid(0.90*z(replication_ratio) + 0.80*z(format_age_hours) - 0.70*z(novelty_score))
saturation_eta_hours = clamp((risk_threshold - saturation_risk) / max(d_risk_dt, 0.001), 6, 168)

Saturation is modeled as pressure over time, not a binary label.

gate health

gate_health = 100 * (
  0.50*gate_pass_rate_24h +
  0.30*velocity_valid_rate_24h +
  0.20*source_coverage
)

Gate health helps prevent false positives from weak or noisy samples.

reception (NLP + local model)

w_i = ln(1 + engagement_i) * exp(-lambda * age_hours_i)
ratio_k = (sum(w_i * p_{i,k}) + alpha_k) / (sum(w_i) + alpha_total)

sentiment_score = ratio_positive - 0.70*ratio_negative
reception_raw = 0.38*sentiment_score + 0.24*(1 - objection_ratio) +
                0.18*proof_demand_ratio + 0.20*purchase_intent_ratio
penalties = 0.12*I(comment_count < min_comments) +
            0.08*max(0, sentiment_floor - sentiment_score)
reception_score = 100 * clamp(reception_raw - penalties, 0, 1)

We use a locally hosted LLM classifier for comment-level probability tags and aggregate with weighted smoothing.

final shootability

shootability_score =
  0.30*momentum_score +
  0.20*breadth_score +
  0.20*(100 - saturation_risk) +
  0.15*gate_health +
  0.15*reception_score

if shootability_score >= 64 and saturation_risk < 72 and gate_health >= 50 => what_to_shoot
if shootability_score in [52,64) => watch
if shootability_score < 52 => do_not_shoot

The verdict combines opportunity and risk, then enforces guardrails.

Worked Example: Sentiment-Led Decision

Illustrative numbers only. This example is intentionally synthetic and safe for public methodology.

input snapshot

  • scan window: 72h
  • comments analyzed: 12,480
  • review mentions: 1,930
  • local-model topic tags per comment: sentiment, objection, proof_demand, purchase_intent

reception calc

ratio_positive = 0.68
ratio_negative = 0.11
objection_ratio = 0.14
proof_demand_ratio = 0.64
purchase_intent_ratio = 0.62

sentiment_score = 0.68 - 0.70*0.11 = 0.603
reception_raw = 0.38*0.603 + 0.24*(1-0.14) + 0.18*0.64 + 0.20*0.62
             = 0.675
penalties = 0.000
reception_score = 67.5

final decision

  • momentum_score = 74.0
  • breadth_score = 66.0
  • saturation_risk = 38.0
  • gate_health = 71.0
  • reception_score = 67.5
shootability_score = 68.6
verdict = what_to_shoot

Interpretation: strong enough to launch controlled tests while saturation risk is still manageable.

Mode Defaults

discovery

Prioritizes momentum and breadth to catch early movement.

exploit

Prioritizes reception quality and proof demand so scale does not collapse.

creator_watch

Prioritizes purchase intent and objection handling for creator selection quality.
